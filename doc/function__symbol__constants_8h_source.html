41  // We do not know whether the compiler constructs the
42  // function symbols first, or whether the function initialise
43  // is called before the constructor. Therefore, we use placement
44  // new as we do not know whether the memory where the elements
45  // are constructed is propertly initialised. This means that the
46  // reference count of these terms can be too high, and these terms
47  // will not property be destroyed, but that has no consequence on
48  // the correctness of the program.
